FileStream

String

DataBase

레거시 Api 중 DataBase는 앱 데이터 폴더 내에 있는 'Database' 폴더에 대한 기능을 제공합니다. 제한범위와 자동 확장자 명시기능 여부에 차이가 있을 뿐, FileStream에서 제공하는 기능과 동일합니다.

이름 형식 설명

getDataBase(fileName: String)

String

앱_데이터_폴더/Database/fileName

파일을 읽어 내용을 문자열로 반환합니다.

setDataBase(fileName: String, data: String)

String

앱_데이터_폴더/Database

폴더 내에

fileName

이라는 파일을 만들고,

data

를 덮어씁니다.

fileName

에 확장자가 명시되지 않은 경우 확장자는 자동으로

.txt

가 됩니다. 최종 파일 내용을 반환합니다.

appendDataBase(fileName: String, data: String)

String

앱_데이터_폴더/Database/fileName

파일에

data

를 이어씁니다. 최종 파일 내용을 반환합니다.

removeDataBase(fileName: String)

Boolean

앱_데이터_폴더/Database/fileName

파일을 삭제합니다.
앱 옵션에서 삭제시 백업기능을 켜두었을 경우,

fileName.bak

으로 내용이 백업됩니다.

java.io.File.delete()

의 결과를 반환합니다.

replier

레거시 Api 중 replier는 메시지 수신에 대한 기능을 제공합니다.

이름 형식 설명

reply(msg: String)

void

 replier객체가 현재 저장하고 있는 방에

reply(room: String, msg: String)

Boolean

room

이라는 방에

msg

라는 내용을 전송합니다.

reply(room: String, msg: String, hideToast: Boolean = false)

Boolean

room

이라는 방에

msg

라는 내용을 전송합니다.
메신저봇R의 작동방식상 메신저봇R이 구동된 이후에 알림을 단 한번도 받지 못한 채팅방으로는 답장이 불가능한데, 이를 시도할 경우 화면 하단에 Toast알림을 표시하게 됩니다. (

hideToast = false

인 경우.)
반환값은 메신저봇이 해당 방의 답장 세션을 가지고 있는지에 대한 여부입니다. 이는 일반적으로 메시지 전송요청의 성공 여부를 나타냅니다.
(단, 통신 문제등으로 전송이 실패하는 경우는 고려하지 않습니다.)
